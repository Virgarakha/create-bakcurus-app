==================================================
1. APA ITU BACKURUS
==================================================

Backurus adalah framework backend Node.js bergaya Laravel (ES Modules).
Tujuan utamanya: DX yang cepat dan ekspresif untuk membangun REST API.

Yang tersedia di scaffold ini:
- CLI `node urus` (generator, migrate, serve, queue, schedule, test, dll)
- Router `Route.get/post/put/delete` dengan aksi `Controller@method` + middleware alias
- Global error handler (dev/prod) + pretty error page di development
- Response helper konsisten (format success/error)
- Validator advanced (nested, array wildcard, file/image, conditional, sanitization, i18n, custom messages)
- ORM Eloquent-style + query builder + relasi + soft delete
- Migrations + seeders
- JWT auth + middleware `auth`
- Gate/Policy concept
- Cache (memory / redis) + facade/helper
- Event bus + facade/helper
- Queue (sync / redis)
- Scheduler
- WebSocket `/ws`
- Swagger docs route `/docs` dan `/docs.json`
- Helpers: collect/str fluent + macro, app(container), cache/event/log/faker/limit, facades
- Testing helper + test runner (`node urus test`)
- Dev server hot reload dengan restart aman (mencegah EADDRINUSE)

3. MENJALANKAN PROJECT (QUICKSTART)
==================================================

1) Install dependencies:
  npm install

2) Jalankan migration:
  node urus migrate

3) Jalankan seeder:
  node urus db:seed

4) Jalankan development server (watcher + hot reload):
  node urus serve

5) Jalankan server tanpa watcher:
  node index.js

Catatan:
- `node urus serve` menampilkan banner + status loading + ringkasan routes/plugins/modules.
- Hot reload sudah memakai restart aman (stop -> wait -> start) agar tidak terjadi port conflict (EADDRINUSE).

{
  "status": "error",
  "success": false,
  "code": 422,
  "type": "validation",
  "message": "Validation failed",
  "errors": { ... }
}

Helpers response (core/response.js):

- res.paginated(items, meta, message = 'OK')

- res.validationError(errors, message = 'Validation failed')
- res.serverError(message = 'Internal Server Error')
- res.databaseError(message, code = 500)
- res.notFound(message = 'Not Found')
- res.unauthorized(message = 'Unauthorized')
- res.forbidden(message = 'Forbidden')

8. GLOBAL ERROR HANDLING (DEV vs PROD)
==================================================

Semua thrown error ditangani oleh global error handler (core/errors/Handler.js).

Mode development (APP_ENV=development):
- JSON error response menyertakan: file, line, stack
- Jika request Accept `text/html`, akan menampilkan pretty error page.

Mode production (APP_ENV=production):
- Error 5xx disamarkan menjadi "Internal Server Error"
- Error 4xx yang exposed tetap menampilkan message

HTTP error classes (core/errors/HttpError.js):
- HttpError
- NotFoundError
- UnauthorizedError
- ForbiddenError
- ValidationError
- DatabaseError

Contoh:
throw new NotFoundError('User not found')

Logging:
- error handler menulis ke `logs/error.log` untuk error 5xx dan database.

9. VALIDATION (REQUEST, SCHEMA, SANITIZE)
==================================================

Backurus punya validator advanced di `core/validator.js`.

A) Request Class (seperti Laravel FormRequest)
class StoreUserRequest {
  rules() {
    return {
      name: 'required|alpha_spaces|min:3',
      email: 'required|email',
      password: 'required|password'
    }
  }

  messages() {
    return {
      email: {
        required: 'Email wajib diisi',
        email: 'Format email tidak valid'
      }
    }
  }

  sanitize() {
    return {
      name: 'trim',
      email: 'trim|lowercase'
    }
  }
}

Pakai di controller:
  await validate(req, StoreUserRequest)

Atau auto di route:
  Route.post('/users', 'UserController@store').validate(StoreUserRequest)

B) Schema Validation langsung (data + rules)
  await validate(data, {
    'user.name': 'required|min:3',
    'user.email': 'required|email'
  })

C) Nested object & array wildcard
Nested:
  validate(data, { 'user.name': 'required' })

Array wildcard:
  validate(data, { tags: 'array', 'tags.*': 'string|min:2' })

D) Conditional rules
- required_if:role,admin
- required_unless:role,admin
- required_with:fieldA,fieldB
- required_without:fieldA,fieldB

E) File & image validation (multipart/form-data)
File object dari `core/request.js` berbentuk:
  { originalName, mimetype, size, buffer }

Rules:
- file
- mimes:jpg,png
- image
- dimensions:min_width=100,min_height=100
- min / max (untuk file dihitung dalam KB jika file-like)

Contoh:
  await validate({}, { avatar: 'required|file|mimes:png,jpg|max:2048' }, null, { files: req.files })

F) Localization
File i18n:
  resources/lang/en/validation.js
  resources/lang/id/validation.js

Set locale:
  APP_LOCALE=id

G) Sanitization helper
  const clean = sanitize(data, { name: 'trim', username: 'slug' })

H) Hooks style fluent
  await validator(data, rules)
    .before((data) => { data.email = String(data.email || '').trim() })
    .sanitize({ email: 'lowercase' })
    .after((_data, { errors }) => { /* logging */ })
    .validate()

26. TROUBLESHOOTING
==================================================

1) Port conflict (EADDRINUSE)
- Hot reload sudah restart aman.
- Jika port masih bentrok, cek ada process lain memakai APP_PORT.

2) Error DB connect
- Pastikan MySQL/Redis hidup dan `.env` benar.

3) Error response tidak detail
- Pastikan APP_ENV=development.
